# River Reviewer のビジョン

> **内部資料:** 設計思想のドラフトです。公開ドキュメントは [`pages/explanation/`](../pages/explanation/) を参照してください。

River Reviewer は、LLM を使ったコードレビューツールではなく、ソフトウェア開発におけるレビュー職務をスキルとして定義・評価・運用するための基盤である。

## 最終像

スキルを「単発の能力」や「プロンプト」ではなく、観測可能・評価可能・再利用可能な仕事の単位として扱う。
人間のレビュー職務と同じ水準で、責任・判断・成果条件を明示し、継続的に育てられる状態を作る。

## スキルの再定義（職務単位）

- スキルは「レビュー工程」ではなく「レビュー職務」を代替・補完する役割として設計する。
- 何を責任として引き受け、どの判断を自動で行い、どこから人間へ返すかを明確にする。
- 成果が「良いレビューだった」と判断できる条件を持つ。

## agentskills.io から取り入れる視点

- **観測可能性**: 実行結果と根拠を追跡できるようにする。
- **評価可能性**: ルーブリック / 指標で合否や改善方向を判断できるようにする。
- **再利用可能性**: スキルを組織内外で再利用できる設計にする。

## Anthropic Skills との差分

- Anthropic Skills は「業務マニュアル付きの道具箱」。
- River Reviewer は、レビュー職務をスキルとして定義し、評価制度と運用を含めた「職務基盤」として位置づける。

## 責任境界と人間中心の原則

- スキルは自律的に暴れる存在ではなく、責任境界が明確な労働者として扱う。
- 迷いがあるときは人間に返すシグナルを持たせる。
- 最終判断は人間が行う。

## 拡張可能性

- レビュー以外（ADR、設計、運用、SRE）にも横展開できる設計を想定する。
- 「良いツール」ではなく「良い仕事の作り方」を実装する方向へ進む。
