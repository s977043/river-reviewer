## Stop-on-Stop-Judge 運用ルール

「対応を自律的に進めて」は無制限委任ではなく、**既知の目的に対する実行許可** として扱う。Codex stop judgment 後は以下 3 択で分類。

| バケット     | 該当例                                                         | 今回該当                                                                              |
| ------------ | -------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| **Continue** | production bug / failed CI / release blocker / user 明示タスク | A2-fix-4 (riskAssessment), brace-expansion CVE close, risk-map example, smoke runbook |
| **Consult**  | scope expansion / 新規 convention / 追加 skill / docs 体系変更 | community skill #884, registry 設計, A2-3 着手                                        |
| **Park**     | nice-to-have / 品質差が測れない改善 / eval なし追加            | docs about docs, marginal skill polish, README drift fix の連続                       |

このルールは [`AGENTS.md`](../../../AGENTS.md) の autonomous-loop 運用ガイドラインに反映予定（別 PR）。

### 再開条件 (Stop 解除トリガー)

Stop 判定は無期限ではなく、以下のいずれかが成立したら Codex に再判定を依頼して解除可否を決める。「対応を自律的に進めて」は **再判定トリガーではあるが解除条件ではない**（再判定で Stop 継続と判断されることもある）。

1. **新情報の到達**: production bug / 新規 CVE / failed CI / 外部からの blocker 報告
2. **明示的な目的変更**: ユーザーが具体的な Issue / PR / scope を指定（"自律的に進めて" だけでは不足）
3. **リスク低下**: 検証基盤の整備（fixtures / golden output / smoke test）が完了し、scope creep の懸念が解消した
4. **時間経過 + 観察結果**: adopter からのフィードバックなど、observation phase で測ろうとした data が取れた

### Stop phase 中に許容される行為

- 状態確認（残 issue / open PR / CI 状態のステータス報告）
- レビュー / 監査 / 振り返り
- ドキュメント参照
- **新規 PR 作成・コード変更は禁止**（再開条件成立後のみ）

### Stop phase の Operating Mode (soft-violation gray zone の解消)

実運用で「Stop 中の status 報告は OK か」が曖昧になり soft violation を生んだため、以下を明文化。

| 行為                                          | 判定                            | 補足                                                               |
| --------------------------------------------- | ------------------------------- | ------------------------------------------------------------------ |
| 単発のステータス確認                          | OK                              | 「open PR / issue / CI green か」程度                              |
| 振り返り / 多視点レビュー / 監査              | **推奨**                        | Stop は観察フェーズでもある                                        |
| Adopter / 外部からのフィードバック観察        | **推奨**                        | 「時間経過 + 観察結果」再開条件のための data 収集                  |
| 連続したステータス報告 + 候補列挙             | NG (soft violation)             | 再開を間接示唆する。ユーザーが scope 指定したらのみ列挙            |
| ドキュメント参照 / コード read                | OK                              | 学習・準備として正当                                               |
| 「ついで」のドキュメント整理 / refactor       | NG                              | scope 外。再開条件 #2 を経由していない                             |
| 振り返りから派生する meta-PR (rule codify 等) | **再開条件 #1 (新情報) で正当** | 振り返り自体が新情報源。ただし Codify-then-validate を必ず適用する |

### 解除手順

1. 再開条件のうちどれが成立したかを明示
2. Codex に「Stop 継続 vs 解除」の判定を依頼
3. 解除する場合は具体的な scope（着手 Issue 番号 + 完了条件）を 1 つ宣言してから再開
