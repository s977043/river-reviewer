# AI レビュー調整ログ（Dynamic）

## 目的
- AIレビューサービス / River-Reviewer のプロンプト・設定調整を記録し、再現性を確保する。
- 調整結果を Static へ昇格するか判断する。

## 前提（参考記事の示唆）
- 「コメントしない」指示を入れても **一定割合でコメントが出る**ことがある。
- レビューの **スコアが甘めに偏る**傾向がある。
- **詳細レビューより要約が有効**なケースがある。

## 調整の基本方針
- **設定フラグ優先**: 否定命令より `review_comment_lgtm: false` などを先に使う。
- **指示は短く明確**: 長文は効果が不安定になりやすい。
- **検証ループ**: 変更 → 1〜2 週間観測 → 結果記録 → 反映/撤回。

## 記録テンプレート
- 日付:
- 目的/仮説:
- 変更内容:
- 期待する効果:
- 結果（定量/定性）:
- 次のアクション:
- Static へ昇格するか: はい/いいえ（理由）

## 例（ダミー）
- 日付: 2025-02-01
- 目的/仮説: LGTM コメントを抑制すればノイズが減る
- 変更内容: `.river-review-bot.yaml` に `review_comment_lgtm: false` を追加
- 期待する効果: コメント数が 30% 以上減る
- 結果（定量/定性）: コメント数が 35% 減少、要約の可読性は維持
- 次のアクション: 重要パスのみ詳細レビューに絞る設定を追加
- Static へ昇格するか: はい（ノイズ抑制の標準化）
