# Documentation policy

このリポジトリのドキュメントは、次の方針で管理します。

## 公開ドキュメントの単一ソース

- 公開ドキュメントの正（Single Source of Truth）は `pages/` とします。
- Web サイトに公開される内容は必ず `pages/` 配下に追加・更新してください。
- 公開ドキュメントの修正 PR では、原則として `pages/` 以外に同内容を複製しません。

## 非公開ドキュメント領域

- `docs/` は公開しない領域として扱います。
- `docs/` には、設計メモ、調査ログ、ADR、運用メモなど内部向け資料のみを置きます。
- 公開が必要な内容は `pages/` に移動し、`docs/` 側は必要に応じて参照リンクに留めます。

### `docs/` の構成（目安）

`docs/` 配下は用途別に整理します。以下は現状の目安であり、**厳密な分類表ではありません**。ツールや開発プロセス由来のディレクトリ（ADR や移行ガイドなど）も必要に応じて追加してよく、迷う場合は本ポリシーの「例外」に従って PR 本文に理由を記載します。

| ディレクトリ        | 用途                                                   |
| ------------------- | ------------------------------------------------------ |
| `docs/adr/`         | Architecture Decision Records                          |
| `docs/development/` | 開発者向けリファレンス・チェックリスト・retrospectives |
| `docs/review/`      | レビュー基準（severity / 観点 / troubleshooting）      |
| `docs/runbook/`     | 運用手順（リリース・smoke など）                       |
| `docs/migration/`   | 移行ガイド                                             |
| `docs/use-cases/`   | ユースケース別の解説                                   |
| `docs/ai/`          | AI / 設計の概観メモ                                    |
| `docs/data/`        | 生成データ（ダッシュボード統計など）                   |

## 例外

- root 直下（README/CONTRIBUTING など）は、プロジェクトの入口・規約・運用ルールとして扱います。
- 例外的な配置が必要な場合は、その理由を PR 本文に記載してください。
