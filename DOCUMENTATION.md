# Documentation policy

このリポジトリのドキュメントは、次の方針で管理します。

## 公開ドキュメントの単一ソース

- 公開ドキュメントの正（Single Source of Truth）は `pages/` とします。
- Web サイトに公開される内容は必ず `pages/` 配下に追加・更新してください。
- 公開ドキュメントの修正 PR では、原則として `pages/` 以外に同内容を複製しません。

## 非公開ドキュメント領域

- `docs/` は公開しない領域として扱います。
- `docs/` には、設計メモ、調査ログ、ADR、運用メモなど内部向け資料のみを置きます。
- 公開が必要な内容は `pages/` に移動し、`docs/` 側は必要に応じて参照リンクに留めます。

## 例外

- root 直下（README/CONTRIBUTING など）は、プロジェクトの入口・規約・運用ルールとして扱います。
- 例外的な配置が必要な場合は、その理由を PR 本文に記載してください。
