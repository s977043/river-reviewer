# フェーズ固有のレビューを実行する

Upstream（上流）、Midstream（中流）、または Downstream（下流）のフィードバックのみが必要な場合に、単一のフェーズをターゲットにします。

## 手順

1. 各スキルの front matter で正しい `phase` をタグ付けする。
2. 必要に応じて CI でのレビュースコープを制限する:

   ```yaml
   on:
     pull_request:
       paths:
         - 'pages/**' # Upstream 向けの例
   ```

3. ローカル実行の場合、ファイルをフィルタリングするか、一時的に `applyTo` グロブを絞り込むことで、関連するスキルのみを実行する。
4. レビュアーがどのチェックを期待すべきか分かるように、PR テンプレートで選択したフェーズを共有する。

## 検証

- レビューアがターゲットセグメントに一致する `phase` のスキルのみを読み込むことを確認する。
- 異なるフェーズに小さな変更を加えて、それが無視されることを確認する。
